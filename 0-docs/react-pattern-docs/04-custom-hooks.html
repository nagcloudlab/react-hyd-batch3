Logic Pattern

Custom Hooks

Extract and reuse stateful logic without changing your component hierarchy

The Problem

Duplicated Stateful Logic Across Components

Multiple components need the same combination of useState and useEffect — for tracking window size, syncing to localStorage, fetching data, or handling form state. Before hooks, the only options were HOCs and render props, both of which add wrapper components to the tree.

Before: Duplicated Window Resize Logic

// --- Navbar.jsx ---
function Navbar() {
  const [windowSize, setWindowSize] = useState({
    width: window.innerWidth,
    height: window.innerHeight,
  });

  useEffect(() => {
    const handleResize = () => {
      setWindowSize({ width: window.innerWidth, height: window.innerHeight });
    };
    window.addEventListener('resize', handleResize);
    return () => window.removeEventListener('resize', handleResize);
  }, []);

  return (
    <nav>
      {windowSize.width > 768 ? <DesktopMenu /> : <MobileMenu />}
    </nav>
  );
}

// --- Sidebar.jsx --- (exact same resize logic!)
function Sidebar() {
  const [windowSize, setWindowSize] = useState({
    width: window.innerWidth,
    height: window.innerHeight,
  });

  useEffect(() => {
    const handleResize = () => {
      setWindowSize({ width: window.innerWidth, height: window.innerHeight });
    };
    window.addEventListener('resize', handleResize);
    return () => window.removeEventListener('resize', handleResize);
  }, []);

  if (windowSize.width < 768) return null;
  return <aside>Sidebar content</aside>;
}

Two components, identical useState + useEffect logic. Every new component that needs window dimensions repeats the same 10 lines.

How Custom Hooks Work

Each component that calls useWindowSize() gets its own independent copy of the state. The logic is shared, but the state is not. No wrapper components are added to the tree.

The Solution

Extract Logic Into a Custom Hook

Move the shared useState + useEffect logic into a function that starts with use. Components call the hook like any built-in hook and receive the values they need.

After: useWindowSize Custom Hook

// --- useWindowSize.js ---
import { useState, useEffect } from 'react';

export function useWindowSize() {
  const [size, setSize] = useState({
    width: window.innerWidth,
    height: window.innerHeight,
  });

  useEffect(() => {
    const handleResize = () => {
      setSize({ width: window.innerWidth, height: window.innerHeight });
    };
    window.addEventListener('resize', handleResize);
    return () => window.removeEventListener('resize', handleResize);
  }, []);

  return size;
}

Components: Clean and Focused

import { useWindowSize } from './useWindowSize';

// --- Navbar.jsx ---
function Navbar() {
  const { width } = useWindowSize();
  return <nav>{width > 768 ? <DesktopMenu /> : <MobileMenu />}</nav>;
}

// --- Sidebar.jsx ---
function Sidebar() {
  const { width } = useWindowSize();
  if (width < 768) return null;
  return <aside>Sidebar content</aside>;
}

Bonus: useLocalStorage — Hooks That Compose

import { useState, useEffect } from 'react';

export function useLocalStorage(key, initialValue) {
  // Initialize state from localStorage (or use the default)
  const [value, setValue] = useState(() => {
    try {
      const stored = localStorage.getItem(key);
      return stored !== null ? JSON.parse(stored) : initialValue;
    } catch {
      return initialValue;
    }
  });

  // Sync to localStorage whenever the value changes
  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(value));
  }, [key, value]);

  return [value, setValue];
}

// Usage — works just like useState, but persists!
function ThemeToggle() {
  const [theme, setTheme] = useLocalStorage('theme', 'light');

  return (
    <button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>
      Current: {theme}
    </button>
  );
}

Rules of Custom Hooks

  • The function name must start with use — this tells React to enforce the Rules of Hooks (no conditional calls, no loops).
  • Custom hooks can call other hooks — including both built-in hooks and other custom hooks. This is how hooks compose.
  • Each call to a custom hook gets its own independent state — calling useWindowSize() in two components creates two separate useState instances. State is never shared between callers.

When to Use

Use Custom Hooks When

  • Two or more components share identical stateful logic (effects, state, subscriptions)
  • You want to test logic independently from the UI
  • A component's logic is complex enough to extract for clarity
  • You want to replace an HOC or render prop with a simpler approach

Avoid Custom Hooks When

  • The function is a pure utility that does not use any hooks — just make it a regular function
  • The hook is overly abstract and hides important details that the component should control
  • Only one component uses it and the logic is simple — extraction adds indirection without benefit

Key Takeaway

Custom Hooks are the modern answer to logic reuse in React. They replace HOCs and Render Props for most use cases while keeping your component tree flat. If you find yourself copying useState + useEffect between components, you have a custom hook waiting to be extracted.
