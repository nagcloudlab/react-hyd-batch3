Performance Pattern

Code Splitting & Lazy Loading

Load code on demand to reduce initial bundle size and improve Time to Interactive

The Problem

The Monolithic Bundle

As your app grows, the JavaScript bundle grows with it. Users download megabytes of code for pages they may never visit. The admin panel, the charts library, the settings page — all loaded upfront even if the user only visits the homepage. This destroys your Time to Interactive and wastes bandwidth.

// Every import is bundled into ONE massive file
import Dashboard from './pages/Dashboard';
import AdminPanel from './pages/AdminPanel';       // 400KB — only for admins
import Settings from './pages/Settings';           // 200KB — rarely visited
import Analytics from './pages/Analytics';         // 300KB — heavy chart library
import UserProfile from './pages/UserProfile';     // 150KB
import Reports from './pages/Reports';             // 250KB — PDF generation

// User visits homepage → downloads ALL of the above (2.5MB+)
function App() {
  return (
    <Routes>
      <Route path="/" element={<Dashboard />} />
      <Route path="/admin" element={<AdminPanel />} />
      <Route path="/settings" element={<Settings />} />
      <Route path="/analytics" element={<Analytics />} />
      <Route path="/profile" element={<UserProfile />} />
      <Route path="/reports" element={<Reports />} />
    </Routes>
  );
}

Before vs. After

The Solution

React.lazy + Suspense

Use React.lazy() to dynamically import components and Suspense to show a fallback while the code loads. The bundler (Webpack, Vite, etc.) automatically creates separate chunks for each lazy import.

Strategy 1: Route-Based Splitting (Most Common)

Split at the route level — each page becomes its own chunk, loaded only when the user navigates to it.

import { Suspense, lazy } from 'react';
import { Routes, Route } from 'react-router-dom';
import Spinner from './components/Spinner'; // Keep in main bundle

// Each of these creates a separate JS chunk
const Dashboard = lazy(() => import('./pages/Dashboard'));
const AdminPanel = lazy(() => import('./pages/AdminPanel'));
const Settings = lazy(() => import('./pages/Settings'));
const Analytics = lazy(() => import('./pages/Analytics'));
const UserProfile = lazy(() => import('./pages/UserProfile'));
const Reports = lazy(() => import('./pages/Reports'));

function App() {
  return (
    {/* Suspense shows a fallback while the lazy component loads */}
    <Suspense fallback={<Spinner />}>
      <Routes>
        <Route path="/" element={<Dashboard />} />
        <Route path="/admin" element={<AdminPanel />} />
        <Route path="/settings" element={<Settings />} />
        <Route path="/analytics" element={<Analytics />} />
        <Route path="/profile" element={<UserProfile />} />
        <Route path="/reports" element={<Reports />} />
      </Routes>
    </Suspense>
  );
}

Strategy 2: Component-Based Splitting

Lazy load heavy components within a page — useful for tabs, modals, or conditionally shown content.

const HeavyChart = lazy(() => import('./components/HeavyChart'));

function AnalyticsPage() {
  const [activeTab, setActiveTab] = useState('overview');

  return (
    <div>
      <nav>
        <button onClick={() => setActiveTab('overview')}>Overview</button>
        <button onClick={() => setActiveTab('charts')}>Charts</button>
      </nav>

      {activeTab === 'overview' && <Overview />}

      {activeTab === 'charts' && (
        {/* Chart library (300KB) only loads when user clicks the Charts tab */}
        <Suspense fallback={<div>Loading charts...</div>}>
          <HeavyChart data={analyticsData} />
        </Suspense>
      )}
    </div>
  );
}

Strategy 3: Error Handling with Lazy Loading

Network requests can fail. Wrap lazy-loaded components with an error boundary to handle loading failures gracefully.

import { Component, Suspense, lazy } from 'react';

// A reusable error boundary for lazy-loaded components
class LazyErrorBoundary extends Component {
  state = { hasError: false };

  static getDerivedStateFromError() {
    return { hasError: true };
  }

  render() {
    if (this.state.hasError) {
      return (
        <div style={{ padding: '2rem', textAlign: 'center' }}>
          <p>Failed to load this section.</p>
          <button onClick={() => {
            this.setState({ hasError: false });
            window.location.reload();
          }}>
            Retry
          </button>
        </div>
      );
    }
    return this.props.children;
  }
}

// Usage: wrap Suspense with the error boundary
const AdminPanel = lazy(() => import('./pages/AdminPanel'));

function App() {
  return (
    <LazyErrorBoundary>
      <Suspense fallback={<Spinner />}>
        <AdminPanel />
      </Suspense>
    </LazyErrorBoundary>
  );
}

Strategy 4: Library-Based Splitting

Heavy third-party libraries can be split out so they only load when needed.

// Instead of importing the heavy library at the top level...
// import { PDFDocument } from 'pdf-lib'; // 500KB!

// ...import it dynamically only when the user needs it
function ReportsPage() {
  const [generating, setGenerating] = useState(false);

  async function handleGeneratePDF() {
    setGenerating(true);

    // Library is fetched only when this function runs
    const { PDFDocument } = await import('pdf-lib');

    const doc = await PDFDocument.create();
    // ...generate PDF
    setGenerating(false);
  }

  return (
    <button onClick={handleGeneratePDF} disabled={generating}>
      {generating ? 'Generating...' : 'Export PDF'}
    </button>
  );
}

Splitting Strategies Compared

Route-Based

Split at page/route boundaries. Easiest to implement and gives the biggest impact. Start here.

Component-Based

Split heavy components within a page (modals, tabs, below-the-fold content). Good for large pages.

Library-Based

Dynamically import heavy third-party libraries only when the feature that uses them is triggered.

Prefetching

Preload chunks on hover or idle time: import('./Admin') on link hover, so it's ready before the click.

When to Use

Use When

  • Large applications with many routes
  • Routes with heavy components (admin panels, dashboards)
  • Rarely visited pages (settings, reports)
  • Heavy third-party libraries (charts, PDF, editors)
  • You want to improve initial load time

Avoid When

  • Small apps where the total bundle is already small
  • Components that are always visible on initial load
  • Above-the-fold critical content (users see a flash)
  • Micro-optimizing tiny components (overhead not worth it)

Key Takeaway

Code splitting is the single biggest performance win for most React apps. Start with route-based splitting — it's easy to implement and gives the most benefit. Every React.lazy() call tells the bundler "put this in a separate file and load it only when needed." Combine with Suspense for a smooth loading experience, and always add an error boundary for resilience.
