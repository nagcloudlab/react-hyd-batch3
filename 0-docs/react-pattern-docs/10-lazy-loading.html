Performance Pattern

Lazy Loading

Defer loading of off-screen content until the user actually needs it

Lazy loading is a performance strategy that delays the loading or rendering of content until it enters (or is about to enter) the user's viewport. Unlike code splitting (Pattern 08), which focuses on breaking JavaScript bundles into smaller chunks, lazy loading is about deferring visible content — images, components below the fold, infinite scroll lists, and heavy widgets — using the Intersection Observer API.

The Problem

Eagerly Loading Everything Wastes Resources

A page with 50 product cards, each featuring a high-resolution image, loads all 50 images immediately even though the user only sees 4–6 on screen. A dashboard loads heavy chart widgets below the fold that the user hasn't scrolled to yet. This wastes bandwidth, consumes memory, and dramatically slows the initial paint.

Before: Eager Loading (the wasteful way)

// ProductGrid.jsx — loads ALL 50 images immediately on mount

How It Works

Solution 1: Lazy Loading Images

LazyImage Component with IntersectionObserver

Create a reusable component that only loads an image when it scrolls into (or near) the viewport. A placeholder is shown until the image is needed, saving bandwidth and speeding up initial render.

The Native Approach (simplest)

// The easiest win — just add loading="lazy" to your img tags
function ProductCard({ product }) {
  return (
    <div className="product-card">
      <img
        src={product.imageUrl}
        alt={product.name}
        loading="lazy"       // Browser handles lazy loading!
        width={400}            // Always set dimensions to prevent layout shift
        height={300}
      />
      <h3>{product.name}</h3>
      <p>${product.price}</p>
    </div>
  );
}

Custom LazyImage with IntersectionObserver

import { useState, useRef, useEffect } from 'react';

function LazyImage({ src, alt, width, height, placeholder, ...props }) {
  const [isLoaded, setIsLoaded] = useState(false);
  const [isInView, setIsInView] = useState(false);
  const imgRef = useRef(null);

  useEffect(() => {
    const element = imgRef.current;
    if (!element) return;

    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          setIsInView(true);
          observer.unobserve(element); // Stop observing once visible
        }
      },
      {
        rootMargin: '200px', // Start loading 200px before it enters viewport
        threshold: 0,
      }
    );

    observer.observe(element);

    return () => observer.disconnect();
  }, []);

  return (
    <div
      ref={imgRef}
      style={{ width, height, background: '#f1f5f9', borderRadius: '8px' }}
    >
      {isInView ? (
        <img
          src={src}
          alt={alt}
          width={width}
          height={height}
          onLoad={() => setIsLoaded(true)}
          style={{
            opacity: isLoaded ? 1 : 0,
            transition: 'opacity 0.3s ease',
          }}
          {...props}
        />
      ) : (
        placeholder || <div className="image-placeholder" />
      )}
    </div>
  );
}

// Usage:
function ProductGrid({ products }) {
  return (
    <div className="grid">
      {products.map(product => (
        <div key={product.id} className="product-card">
          <LazyImage
            src={product.imageUrl}
            alt={product.name}
            width={400}
            height={300}
          />
          <h3>{product.name}</h3>
          <p>${product.price}</p>
        </div>
      ))}
    </div>
  );
}

Solution 2: Lazy Loading Components on Scroll

useLazyLoad Hook — Render Components Only When Visible

For heavy components like charts, maps, or comment sections that sit below the fold, delay their rendering entirely until the user scrolls to them. This is different from code splitting — the JS is already loaded, but we skip the render work.

Reusable useLazyLoad Hook

import { useState, useRef, useEffect } from 'react';

function useLazyLoad(options = {}) {
  const {
    rootMargin = '100px',  // Start rendering 100px before viewport
    threshold = 0,
    triggerOnce = true,    // Once visible, stay rendered
  } = options;

  const [isVisible, setIsVisible] = useState(false);
  const ref = useRef(null);

  useEffect(() => {
    const element = ref.current;
    if (!element) return;

    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          setIsVisible(true);
          if (triggerOnce) {
            observer.unobserve(element);
          }
        } else if (!triggerOnce) {
          setIsVisible(false);
        }
      },
      { rootMargin, threshold }
    );

    observer.observe(element);

    return () => observer.disconnect();
  }, [rootMargin, threshold, triggerOnce]);

  return { ref, isVisible };
}

export default useLazyLoad;

LazySection Wrapper Component

function LazySection({ children, height = '400px', placeholder, ...options }) {
  const { ref, isVisible } = useLazyLoad(options);

  return (
    <div ref={ref} style={{ minHeight: isVisible ? 'auto' : height }}>
      {isVisible ? (
        children
      ) : (
        placeholder || (
          <div style={{
            height,
            background: '#f1f5f9',
            borderRadius: '12px',
            display: 'flex',
            alignItems: 'center',
            justifyContent: 'center',
            color: '#94a3b8',
          }}>
            Loading when visible...
          </div>
        )
      )}
    </div>
  );
}

// Usage: Dashboard with heavy widgets below the fold
function Dashboard() {
  return (
    <div>
      {/* Above the fold — renders immediately */}
      <MetricsSummary />
      <RecentActivity />

      {/* Below the fold — deferred until scrolled into view */}
      <LazySection height="500px">
        <HeavyChartWidget data={salesData} />
      </LazySection>

      <LazySection height="300px">
        <CommentsSection postId={postId} />
      </LazySection>

      <LazySection height="200px">
        <Footer />
      </LazySection>
    </div>
  );
}

Solution 3: Infinite Scroll / Load More

useInfiniteScroll — Load Data as the User Scrolls

Place an invisible sentinel element at the bottom of a list. When it enters the viewport, fetch the next batch of data. No "Load More" button needed — the experience is seamless. 

import { useState, useRef, useEffect, useCallback } from 'react';

function useInfiniteScroll(fetchMore, options = {}) {
  const {
    rootMargin = '300px',  // Trigger 300px before reaching the end
    threshold = 0,
  } = options;

  const [isLoading, setIsLoading] = useState(false);
  const [hasMore, setHasMore] = useState(true);
  const sentinelRef = useRef(null);

  const handleIntersection = useCallback(
    async ([entry]) => {
      if (entry.isIntersecting && !isLoading && hasMore) {
        setIsLoading(true);
        try {
          const moreAvailable = await fetchMore();
          setHasMore(moreAvailable);
        } finally {
          setIsLoading(false);
        }
      }
    },
    [fetchMore, isLoading, hasMore]
  );

  useEffect(() => {
    const element = sentinelRef.current;
    if (!element) return;

    const observer = new IntersectionObserver(handleIntersection, {
      rootMargin,
      threshold,
    });

    observer.observe(element);
    return () => observer.disconnect();
  }, [handleIntersection, rootMargin, threshold]);

  return { sentinelRef, isLoading, hasMore };
}

export default useInfiniteScroll;

Product List with Infinite Scroll

function ProductList() {
  const [products, setProducts] = useState([]);
  const [page, setPage] = useState(1);
  const PAGE_SIZE = 10;

  const fetchMore = useCallback(async () => {
    const res = await fetch(`/api/products?page=${page}&limit=${PAGE_SIZE}`);
    const newProducts = await res.json();

    setProducts(prev => [...prev, ...newProducts]);
    setPage(prev => prev + 1);

    // Return false when no more data
    return newProducts.length === PAGE_SIZE;
  }, [page]);

  const { sentinelRef, isLoading, hasMore } = useInfiniteScroll(fetchMore);

  return (
    <div>
      <h1>Products</h1>
      <div className="product-grid">
        {products.map(product => (
          <ProductCard key={product.id} product={product} />
        ))}
      </div>

      {/* Invisible sentinel — triggers loading when scrolled near */}
      <div ref={sentinelRef} style={{ height: '1px' }} />

      {isLoading && <p className="loading">Loading more products...</p>}
      {!hasMore && <p className="end">You've reached the end!</p>}
    </div>
  );
}

Popular Libraries Worth Knowing

  • react-intersection-observer — Lightweight hook-based API wrapping IntersectionObserver. Great for most use cases.
  • react-lazyload — Simple component-level lazy loading with scroll and resize event support.
  • react-virtualized / react-window — For extremely long lists, virtualizes DOM nodes so only visible items are rendered.

When to Use & When Not To

When to Use

  • Image-heavy pages (galleries, e-commerce grids)
  • Long scrollable content with sections below the fold
  • Dashboards with heavy chart/map widgets below the fold
  • Infinite scroll lists (feeds, search results)
  • Mobile-first apps where bandwidth and data plans matter

When NOT to Use

  • Above-the-fold critical content — lazy loading hero images hurts LCP!
  • Small pages with little content (overhead not worth it)
  • SEO-critical images — use eager loading so crawlers see them
  • Content that is always visible without scrolling
  • Print stylesheets — all content must be present

Key Takeaway

Lazy loading is about respect for the user's resources — don't load what they can't see. Start with native loading="lazy" for images, use IntersectionObserver for components, and always keep above-the-fold content eager. The best lazy loading is invisible to the user — content appears right when they need it, not a moment too late.
