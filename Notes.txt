javascript features
-------------------

-> single-threaded ( light-weighted process )
-> all IO operations are asynchronous ( non-blocking ) - https://latentflip.com/loupe/
-> dynamically-typed ( no need to declare data-type of variable )
-> functions are first-class citizens ( functions can be treated as data )
-> supports both object-oriented and functional programming paradigms
-> huge ecosystem of libraries and frameworks ( 'npm' - node package manager )

how to run javascript code?
----------------------------

-> using javascript engine ( runtime environment ) - https://en.wikipedia.org/wiki/List_of_JavaScript_engines

1. chrome ( v8 engine  (c++))
2. firefox ( spider monkey engine (c++) )
3. safari ( javascript core engine (c++) )
4. edge ( chakra engine (c++) )
5. using nodejs ( v8 engine + c++ bindings for IO operations )
...

problems working with browser APIs directly ( with plain javascript )
-----------------

-> browser compatibility issues ( different browsers may have different implementations of the same API )
-> memory leaks ( if not handled properly, can lead to performance issues )
-> performance issues ( if not handled properly, can lead to slow rendering and unresponsive UI )
-> too much boilerplate code ( can lead to code duplication and maintenance issues )
-> code re-usability issues ( can lead to code duplication and maintenance issues ) 
.....
